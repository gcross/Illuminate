Detailed Description

These macros define tests and test suites.

Note that in the name displayed to the user all the underscores will be replaced with spaces and then leading and trailing whitespace will be trimmed.

#define DONT_TEST_CASE (   caseName)    DEFINE_TEST_CASE(caseName,boost::make_optional(true))

Defines a test case named caseName, but indicates that the test should be skipped.

This macro is like TEST_CASE, but the test body is ignored and the test is tagged as being skipped.

00001 #include "illuminate.hpp"
00002 
00003 DONT_TEST_CASE(Test) {
00004   FAIL("This test should have been skipped!")
00005 }

Root:
    Test... (skipped)

All tests passed!
#define DONT_TEST_SUITE (   suiteName)    DEFINE_TEST_SUITE(suiteName,boost::make_optional(true))

Defines a test suite named suiteName, but indicates that the suite should be skipped.

This macro is like TEST_SUITE, but all of the children will be skipped (unless individually re-enabled using MUST_TEST_SUITE(suiteName) or MUST_TEST_CASE(caseName).

00001 #include "illuminate.hpp"
00002 
00003 DONT_TEST_SUITE(Parent) {
00004   TEST_CASE(Child1) { FAIL("This test should have been skipped!"); }
00005   TEST_SUITE(Child2) {
00006     TEST_CASE(Grandchild) { FAIL("This test should have been skipped!"); }
00007   }
00008 }

Root:
    Parent:
        Child2:
            Grandchild... (skipped)
        Child1... (skipped)

All tests passed!
#define MUST_TEST_CASE (   caseName)    DEFINE_TEST_CASE(caseName,boost::make_optional(false))

Defines a test case named caseName, and runs it even if its parent is flagged to be skipped.

This macro is like TEST_CASE, but it overrides the skip flag of its parent to make sure the test body is run.

00001 #include "illuminate.hpp"
00002 
00003 DONT_TEST_SUITE(Parent) {
00004   MUST_TEST_CASE(Child) {}
00005 }

Root:
    Parent:
        Child... PASSED :-)

All tests passed!
#define MUST_TEST_SUITE (   suiteName)    DEFINE_TEST_SUITE(suiteName,boost::make_optional(false))

Defines a test suite named suiteName, and runs it even if its parent is flagged to be skipped.

This macro is like TEST_SUITE, but it overrides the skip flag of its parent to make sure the test suite is run.

00001 #include "illuminate.hpp"
00002 
00003 DONT_TEST_SUITE(Parent) {
00004   MUST_TEST_SUITE(Child) {
00005     TEST_CASE(Grandchild) {}
00006   }
00007 }

Root:
    Parent:
        Child:
            Grandchild... PASSED :-)

#define TEST_CASE (   caseName)    DEFINE_TEST_CASE(caseName,boost::none)

Defines a test case named caseName.

The contents of the test should be supplied within curly brackets ({}'s) immediately following the macro as in the following example:

00001 #include "illuminate.hpp"
00002 
00003 TEST_CASE(Test) {
00004   ASSERT_EQ_VAL(1+1,2)
00005 }

Root:
    Test... PASSED :-)

#define TEST_SUITE (   suiteName)    DEFINE_TEST_SUITE(suiteName,boost::none)

Defines a test suite named suiteName.

Suites may contain a mixture of test cases and nested test suites in any order. The contents of the suite should be supplied within curly brackets ({}'s) immediately following the macro as in the following example:

00001 #include "illuminate.hpp"
00002 
00003 TEST_SUITE(Parent) {
00004   TEST_CASE(Child1) {}
00005   TEST_SUITE(Child2) {
00006     TEST_SUITE(Grandchild21) {
00007       TEST_CASE(Greatgrandchild211) {}
00008       TEST_SUITE(Greatgrandchild212) {}
00009     }
00010     TEST_CASE(Grandchild22) {}
00011   }
00012 }

Root:
    Parent:
        Child2:
            Grandchild21:
                Greatgrandchild211... PASSED :-)
            Grandchild22... PASSED :-)
        Child1... PASSED :-)

All tests passed!
